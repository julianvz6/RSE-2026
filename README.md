# Individual Assessment 1 - Grobid Pipeline

[![DOI](https://zenodo.org/badge/1154798578.svg)](https://doi.org/10.5281/zenodo.18966270) 
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Automated pipeline that extracts structured data from academic PDF articles using [Grobid](https://github.com/kermitt2/grobid) and produces visualizations.

## Author

- **Julian Alberto Villafuerte Zevallos** — [@julianvz6](https://github.com/julianvz6)

## Features

| Output | Description |
|---|---|
| **Keyword Cloud** | Word cloud generated from all paper abstracts |
| **Figures Chart** | Bar chart showing the number of figures per article |
| **Links List** | All external links found in each paper |

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/julianvz6/RSE-2026.git
```

### 2. Create and activate a virtual environment

```bash
python -m venv venv01
```

**Windows (PowerShell):**
```bash
.\venv01\Scripts\Activate.ps1
```

**Windows (CMD):**
```bash
.\venv01\Scripts\activate.bat
```

**Linux / macOS:**
```bash
source venv01/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Grobid server

A running [Grobid](https://github.com/kermitt2/grobid) server is required to process PDFs. You can run one locally with Docker:

```bash
docker run --rm -p 8070:8070 lfoppiano/grobid:0.8.1
```

Then update `config.json` with your server address (defaults to `http://localhost:8070`). Pre-processed XML files are already included in `grobid_output/`, so this step is optional if you just want to run the analysis.

## Usage

### Option 1: Run locally

```bash
python main.py
```

All paths and settings are read from `setup.cfg`.

### Option 2: Deploy with Docker Compose 

This is the easiest way to run everything — it starts the Grobid server and the pipeline automatically:

```bash
docker compose up --build
```

The compose file defines two services:

| Service | Description |
|---|---|
| **grobid** | Runs the Grobid server (`lfoppiano/grobid:0.8.1`) on port `8070` with a TCP healthcheck |
| **pipeline** | Builds from the `Dockerfile`, waits for Grobid to be healthy, then runs `main.py` |

The pipeline container uses `depends_on` with `condition: service_healthy` to ensure Grobid is fully ready before processing starts. The `GROBID_SERVER` environment variable is automatically set to point to the Grobid container.

Results are persisted to the host via volume mounts (`prac_output/` and `grobid_output/`).

To stop and clean up:

```bash
docker compose down
```

### Output

The pipeline generates the following files in `prac_output/`:

- `wordcloud.png` — keyword cloud from paper abstracts
- `figures_per_article.png` — bar chart of figure counts
- `links_per_paper.txt` — list of links found in each paper

## Configuration

Edit `setup.cfg` to change paths or Grobid settings:

```ini
[paths]
input = ./articles
grobid_output = ./grobid_output
results = ./prac_output

[grobid]
config = ./config.json
workers = 4
force = true
```

## Testing

Unit tests are located in the `tests/` folder and use [pytest](https://docs.pytest.org/). To run them:

```bash
pip install pytest
python -m pytest tests/ -v
```

The test suite covers:

| Test class | What it tests |
|---|---|
| `TestParseGrobidXml` | XML parsing — abstracts, figure counts, links, edge cases |
| `TestGenerateWordcloud` | Word cloud image generation |
| `TestPlotFigureCounts` | Bar chart image generation |
| `TestSaveLinks` | Links text file output and formatting |
